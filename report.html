一、实验目的

本实验旨在通过一个完整的Web应用项目开发，使学生深入理解并掌握以下核心能力：

  1. 掌握Flask框架在项目开发中的应用流程：通过从零构建一个功能完整的Web应用，理解Flask的路由机制、模板渲染、表单处理、数据库ORM等核心概念，并能够熟练运用Flask扩展生态（Flask-SQLAlchemy、Flask-Login、Flask-Migrate等）完成实际开发任务。
  2. 理解Web应用开发的系统工程方法：从需求分析、系统设计、数据库建模、编码实现到测试部署，完整经历软件工程的各个阶段，建立系统化的开发思维。
  3. 熟练掌握Flask实现基本功能模块：包括用户认证（注册/登录/登出）、数据增删改查（CRUD）、表单验证、权限控制、分页查询等Web应用常见功能模块的实现。
  4. 强化工程化开发意识：学习并实践项目结构组织、配置管理、环境隔离、数据库迁移、自动化测试等工程化开发规范，养成良好的编码习惯。
  5. 培养团队协作和文档撰写能力：通过分组协作开发，学习版本控制、任务分工、接口约定等协作技巧，同时培养技术文档的撰写与表达能力。

4.4 关系说明

  • User → Registration：一对多关系。一个用户可以报名多个活动。db.relationship('Registration', backref='user', lazy='dynamic')
  • Event → Registration：一对多关系。一个活动可以有多条报名记录。db.relationship('Registration', backref='event', lazy='dynamic', cascade='all, delete-orphan')
  • User → Event：一对多关系（创建者）。一个用户可以创建多个活动。db.relationship('User', backref='created_events')
  • User ↔ Event：通过Registration表实现多对多关系，报名状态和签到信息存储在Registration中。

4.5 约束说明

  • UniqueConstraint('user_id', 'event_id')：确保同一用户对同一活动只能有一条报名记录，防止重复报名。
  • cascade='all, delete-orphan'：Event删除时，关联的Registration记录自动级联删除，避免孤立数据。
  • UNIQUE约束：User表的username和email字段均设置唯一约束，防止重复注册。
  • NOT NULL约束：关键业务字段（如username、email、password_hash、title、start_time等）不允许为空。

5.2 应用工厂函数 create_app()

采用Application Factory模式，支持创建不同配置的应用实例（开发/测试），便于测试和部署。

# app/__init__.py
import os
import traceback as tb
from flask import Flask, redirect, render_template, request, url_for
from werkzeug.exceptions import HTTPException
from config import Config
from app.extensions import csrf, db, login_manager, migrate, toolbar
from app.logging import setup_logging, register_request_hooks

BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))

def create_app(config_class=Config):
    app = Flask(
        __name__,
        template_folder=os.path.join(BASE_DIR, "templates"),
        static_folder=os.path.join(BASE_DIR, "static"),
    )
    app.config.from_object(config_class)

    # 初始化扩展
    db.init_app(app)
    login_manager.init_app(app)
    migrate.init_app(app, db)
    csrf.init_app(app)
    if app.debug and toolbar is not None:
        toolbar.init_app(app)

    # 初始化日志
    setup_logging(app)
    register_request_hooks(app)

    # 注册蓝图
    from app.auth import auth_bp
    from app.event import event_bp
    from app.checkin import checkin_bp
    from app.admin import admin_bp

    app.register_blueprint(auth_bp)
    app.register_blueprint(event_bp)
    app.register_blueprint(checkin_bp)
    app.register_blueprint(admin_bp)

    # 错误处理
    @app.errorhandler(404)
    def not_found(e):
        return render_template("errors/404.html"), 404

    @app.errorhandler(500)
    def internal_error(e):
        trace = None
        if app.debug:
            trace = "".join(tb.format_exception(type(e), e, e.__traceback__))
        return render_template("errors/500.html", traceback=trace), 500

    @app.route('/')
    def index():
        return redirect(url_for('event.list_events'))

    return app

    registrations = db.relationship('Registration', backref='user', lazy='dynamic')

    def set_password(self, password):
        self.password_hash = generate_password_hash(password)

    def check_password(self, password):

5.5 Event 模型

Event模型通过@property装饰器封装报名人数、签到人数等计算逻辑，保持模板代码简洁。

# app/models.py (Event部分)
class Event(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(200), nullable=False)
    description = db.Column(db.Text)
    location = db.Column(db.String(200))
    start_time = db.Column(db.DateTime, nullable=False)
    end_time = db.Column(db.DateTime, nullable=False)
    max_participants = db.Column(db.Integer, default=0)
    checkin_code = db.Column(db.String(50))
    category = db.Column(db.String(50), default='其他')
    creator_id = db.Column(db.Integer, db.ForeignKey('user.id'), nullable=False)
    created_at = db.Column(db.DateTime, default=lambda: datetime.now(timezone.utc))

    creator = db.relationship('User', backref='created_events')
    registrations = db.relationship('Registration', backref='event',
                                    lazy='dynamic', cascade='all, delete-orphan')

    @property
    def registered_count(self):
        return self.registrations.filter_by(status='confirmed').count()

    @property
    def is_full(self):
        if self.max_participants == 0:
            return False
        return self.registered_count >= self.max_participants

5.6 注册路由

注册流程：验证表单 → 检查用户名/邮箱唯一性 → 创建用户 → 重定向到登录页。密码通过set_password()哈希后存储。

# app/auth/routes.py
@auth_bp.route('/register', methods=['GET', 'POST'])
def register():
    if current_user.is_authenticated:
        return redirect(url_for('event.list_events'))
    form = RegisterForm()
    if form.validate_on_submit():
        if User.query.filter_by(username=form.username.data).first():
            flash('用户名已存在', 'danger')
            return render_template('auth/register.html', form=form)
        if User.query.filter_by(email=form.email.data).first():
            flash('邮箱已被注册', 'danger')
            return render_template('auth/register.html', form=form)
        user = User(username=form.username.data, email=form.email.data)
        user.set_password(form.password.data)
        db.session.add(user)
        db.session.commit()
        flash('注册成功，请登录', 'success')
        return redirect(url_for('auth.login'))
    return render_template('auth/register.html', form=form)

5.7 登录路由

登录流程：查询用户 → 校验密码 → 调用login_user()建立会话 → 重定向到next参数或首页。

# app/auth/routes.py
@auth_bp.route('/login', methods=['GET', 'POST'])
def login():
    if current_user.is_authenticated:
        return redirect(url_for('event.list_events'))
    form = LoginForm()
    if form.validate_on_submit():
        user = User.query.filter_by(username=form.username.data).first()
        if user is None or not user.check_password(form.password.data):
            flash('用户名或密码错误', 'danger')
            return render_template('auth/login.html', form=form)
        login_user(user)
        next_page = request.args.get('next')
        return redirect(next_page or url_for('event.list_events'))
    return render_template('auth/login.html', form=form)

5.8 报名路由

报名逻辑处理三种情况：已取消的重新报名、已报名的提示、新报名。同时校验名额上限。

# app/event/routes.py
@event_bp.route('/<int:id>/register', methods=['POST'])
@login_required
def register(id):
    event = db.get_or_404(Event, id)
    existing = Registration.query.filter_by(
        user_id=current_user.id, event_id=event.id
    ).first()
    if existing:
        if existing.status == 'cancelled':
            if event.is_full:
                flash('名额已满，无法重新报名', 'warning')
                return redirect(url_for('event.detail', id=event.id))
            existing.status = 'confirmed'
            db.session.commit()
            flash('已重新报名', 'success')
        else:
            flash('您已报名此活动', 'info')
    else:
        if event.is_full:
            flash('名额已满，无法报名', 'warning')
            return redirect(url_for('event.detail', id=event.id))
        reg = Registration(user_id=current_user.id, event_id=event.id)
        db.session.add(reg)
        db.session.commit()
        flash('报名成功', 'success')
    return redirect(url_for('event.detail', id=event.id))

    event = db.get_or_404(Event, event_id)

    reg = Registration.query.filter_by(
        user_id=current_user.id, event_id=event_id, status='confirmed',
    ).first()

    if not reg:
        flash('你尚未报名此活动，无法签到', 'warning')
        return redirect(url_for('event.detail', id=event_id))

    if reg.checked_in:
        flash('你已经签到过了', 'info')
        return redirect(url_for('event.detail', id=event_id))

    form = CheckinForm()
    if form.validate_on_submit():
        if hmac.compare_digest(form.code.data.strip(), event.checkin_code):
            reg.checked_in = True
            reg.checked_in_at = datetime.now(timezone.utc)
            db.session.commit()
            flash('签到成功！', 'success')
            return redirect(url_for('event.detail', id=event_id))
        else:
            flash('签到码错误', 'danger')

    return render_template('checkin/checkin.html', form=form, event=event)

def admin_required(f):
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if not current_user.is_admin:
            abort(403)
        return f(*args, **kwargs)
    return decorated_function

def get_qr_code(url):
    """生成QR码并返回base64编码的PNG图片"""

# app/admin/routes.py
import csv, io
from flask import Response
from urllib.parse import quote

@admin_bp.route('/export/<int:event_id>')
def export_event(event_id):
    event = db.session.get(Event, event_id)
    if not event:
        return 'Event not found', 404

    registrations = Registration.query.filter_by(
        event_id=event_id, status='confirmed'
    ).all()

    output.write('\ufeff')
    writer = csv.writer(output)
    writer.writerow(['username', 'email', 'registered_at', 'checked_in', 'checked_in_at'])

    for r in registrations:
        writer.writerow([r.user.username, r.user.email,
            r.registered_at.strftime('%Y-%m-%d %H:%M:%S') if r.registered_at else '',
            'Yes' if r.checked_in else 'No',
            r.checked_in_at.strftime('%Y-%m-%d %H:%M:%S') if r.checked_in_at else ''])

    filename = f'{event.title}_registrations.csv'
    encoded = quote(filename)
    return Response(output.getvalue(),
        mimetype='text/csv; charset=utf-8',
        headers={'Content-Disposition': f"attachment; filename=\"{filename}\"; filename*=UTF-8''{encoded}"})

6.4 毛玻璃效果实现原理


毛玻璃效果通过CSS backdrop-filter属性实现，核心代码如下：


.glass-card {
    background: var(--glass-bg);            /* 半透明背景 */
    backdrop-filter: blur(var(--glass-blur));             /* 核心：背景模糊 */
    -webkit-backdrop-filter: blur(var(--glass-blur));    /* Safari兼容 */
    border: 1px solid var(--glass-border);  /* 细边框增加立体感 */
    border-radius: var(--glass-radius);        /* 20px 统一圆角 */
    box-shadow: 0 8px 32px rgba(0,0,0,0.1);  /* 柔和阴影 */
}


工作原理：backdrop-filter: blur(var(--glass-blur))会对元素背后的内容应用高斯模糊，配合半透明背景色，呈现出类似磨砂玻璃的视觉效果。



6.4b 卡片组件样式简化


卡片组件（.card.glass）移除了装饰性边框和伪元素，采用更简洁的视觉风格：border: none 替代 1px 边框，不再使用 ::before（顶部折射光线）和 ::after（内部折射渐变）伪元素，保留毛玻璃背景模糊和柔和阴影。

十、思考题答案





Q1：需求分析的重要性？如何避免需求不明确？


需求分析是软件开发的基石。在本项目开发初期，我们通过需求分析明确了"活动报名+签到"的核心业务流程，避免了开发过程中的方向偏离。如果需求分析不充分，可能导致开发出的功能与用户实际需求不匹配，造成返工和资源浪费。


避免需求不明确的方法包括：（1）用户故事驱动：以"作为XX，我希望能XX，以便XX"的格式描述需求，确保每个功能都有明确的使用者和使用场景；（2）原型验证：在编码前制作低保真原型，让团队成员和潜在用户提前体验交互流程；（3）迭代细化：采用敏捷开发方式，将大需求拆分为小的用户故事，每个迭代聚焦核心功能，逐步完善；（4）文档化：将需求以结构化文档的形式记录，包括功能需求表、非功能需求、用户角色定义等，作为开发的参考基准。



Q2：为何不用app.run()作为生产服务器？WSGI服务器的作用？


Flask内置的开发服务器（app.run()）不适合生产环境。Flask官方文档明确指出，内置服务器是为开发便利而设计的，默认单线程处理请求，性能和安全性均无法满足生产需求。它不支持并发处理、缺乏安全加固（如HTTPS、请求超时），且未经过安全审计。


WSGI服务器（如Gunicorn、uWSGI）是生产环境的标准选择。WSGI（Web Server Gateway Interface）定义了Python Web应用与Web服务器之间的标准接口。WSGI服务器的作用包括：（1）并发处理：通过多worker进程/线程同时处理多个请求，提升吞吐量；（2）进程管理：自动管理worker进程的生命周期，支持优雅重启；（3）安全隔离：将应用进程与HTTP请求处理分离，减少攻击面；（4）反向代理集成：通常配合Nginx使用，Nginx处理静态文件和负载均衡，WSGI服务器运行Python应用。


生产部署示例：gunicorn -w 4 -b 0.0.0.0:8000 "app:create_app()"



Q3：如何通过Flask实现用户认证和权限管理？


Flask-Login是Flask生态中最常用的认证扩展。在本项目中，用户认证的实现分为以下几个层面：


（1）会话管理：Flask-Login通过login_user(user)将用户ID存储在安全的Cookie中，后续请求通过@login_required装饰器验证用户是否已登录。current_user代理对象提供对当前登录用户的全局访问。


（2）密码安全：使用Werkzeug的generate_password_hash()对密码进行PBKDF2哈希处理后存储，验证时使用check_password_hash()比对，避免明文存储密码。


（3）权限管理：通过User模型的is_admin字段区分角色，配合自定义的@admin_required装饰器实现基于角色的访问控制（RBAC）。管理蓝图使用before_request钩子统一拦截非管理员请求。


（4）CSRF防护：Flask-WTF的CSRFProtect为所有POST表单自动生成和验证CSRF Token，防止跨站请求伪造攻击。



Q4：访问量增大时如何优化Flask应用性能？


当访问量增大时，可从以下几个层面优化Flask应用性能：


（1）数据库优化：为高频查询字段（如Registration.user_id、Registration.event_id）添加数据库索引；使用joinedload()或subqueryload()预加载关联数据，减少N+1查询问题；对复杂统计查询使用原生SQL或数据库视图。


（2）缓存策略：引入Redis或Memcached作为缓存层，缓存活动列表、用户会话等高频读取数据；使用Flask-Caching扩展实现视图级缓存；对静态资源配置CDN和浏览器缓存。


（3）应用服务器优化：使用Gunicorn等WSGI服务器增加worker数量；配合Nginx作为反向代理，处理静态文件和负载均衡；使用异步框架（如Quart）或Celery处理耗时任务。


（4）架构层面：引入读写分离（主从数据库）；对大表进行分表或分区；使用消息队列解耦高并发写操作；考虑容器化部署（Docker + Kubernetes）实现水平扩展。
