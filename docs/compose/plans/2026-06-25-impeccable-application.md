# Impeccable Skill Application Plan

## Applicable Commands

| Command | Target | Why | Priority |
|---------|--------|-----|----------|
| `audit` | report.html + Flask app templates | Technical quality: a11y, performance, responsive | P0 |
| `critique` | Flask app UI | UX design review: hierarchy, clarity | P0 |
| `polish` | report.html + Flask app | Final pass before Friday presentation | P0 |
| `harden` | Flask app | Error handling, i18n edge cases, text overflow | P1 |
| `typeset` | report.html | Typography hierarchy, font choices | P1 |
| `layout` | Both | Spacing, rhythm, visual hierarchy | P1 |
| `clarify` | Flask app | UX copy, labels, error messages | P2 |
| `adapt` | Both | Responsive on mobile | P2 |

## Execution Order

1. **audit** → find issues first
2. **critique** → UX review
3. **fix** all findings
4. **polish** → final pass
5. **harden** → edge cases
6. Review → commit

## Note

Impeccable requires PRODUCT.md and DESIGN.md. The project already has DESIGN.md. Need to create PRODUCT.md first via `/impeccable init` or manually.
